# FocusGuard 집중 강제·입력 안내 설계

## 목표

집중 세션 중 허용되지 않은 프로그램으로 전환하면 허용 화면으로 돌아올 때까지 경고음을 반복하고, 마지막으로 사용했던 허용 프로그램을 자동 복원한다. 메인 화면의 빈 입력칸에는 흐릿한 안내 문구를 표시한다. Google Apps Script를 사용하는 원본 프로젝트와 로컬 번역 배포판에 동일한 동작을 제공한다.

## 적용 범위

- 원본 프로젝트 `src/FocusGuard`
- 로컬 배포 프로젝트 `src/FocusGuard.Local`
- 공용 `WindowManager`는 원본에 구현하고 로컬 프로젝트가 연결해 사용한다.
- 원본과 로컬에 별도로 존재하는 `StudyWindow.xaml.cs`에는 같은 집중 강제 흐름을 반영한다.

## 입력 안내 문구

메인 화면의 빈 입력칸 위에 다음 안내를 흐릿하게 표시한다.

- 과제 입력: `과제명`
- 리소스 입력: `URL 또는 파일 경로`

안내 문구는 마우스 입력을 가로채지 않는다. 입력값이 비어 있을 때만 표시되며, 사용자가 한 글자라도 입력하면 즉시 사라지고 다시 비우면 나타난다. 원본 `MainWindow.xaml`을 로컬 프로젝트도 연결해 사용하므로 한 번의 XAML 변경으로 양쪽에 적용한다.

## 허용 창 추적

집중 단계에서 1초마다 현재 전경 창을 확인한다. 활성 창이 등록된 허용 프로그램 또는 집중용 WebView이면 해당 창 핸들을 마지막 허용 복귀 대상으로 저장한다.

FocusGuard 위젯, 메모 창, 사전·번역 창은 정상적인 보조 동작으로 허용하지만 마지막 외부 허용 창 기록을 덮어쓰지 않는다. 따라서 보조 창을 사용한 뒤 다른 프로그램으로 빠져도 원래 공부하던 앱이나 WebView로 돌아간다.

허용 프로그램이 여러 개라면 가장 최근에 활성화했던 허용 창을 사용한다. 아직 기록된 창이 없다면 등록 순서상 첫 번째로 실행 중인 허용 프로그램을 초기 복귀 대상으로 사용한다.

## 딴짓 감지와 자동 복구

휴식 단계에서는 기존처럼 감시와 강제 복구를 수행하지 않는다. 집중 단계에서 허용되지 않은 창이 활성화되면 한 번의 타이머 처리 안에서 다음을 수행한다.

1. 딴짓 시간을 1초 증가시킨다.
2. 상태 문구를 `딴짓 감지됨`으로 바꾼다.
3. Windows 경고음을 한 번 재생한다.
4. 마지막 허용 창을 복원하고 전면 활성화를 시도한다.

다음 타이머 때도 허용 창이 활성화되지 않았다면 같은 동작을 반복한다. 따라서 복구될 때까지 약 1초 간격으로 경고음이 계속 난다. 자동 복구가 성공해 허용 창이 활성화되면 경고음 반복은 자연스럽게 멈춘다.

최소화된 허용 창은 먼저 복원한다. 일반 권한 앱에서 전면 전환이 실패하면 창의 z-order를 갱신한 뒤 다시 활성화한다. 관리자 권한으로 실행된 대상처럼 Windows 보안 경계 때문에 강제 활성화가 불가능한 경우에는 앱을 종료하거나 권한을 우회하지 않고, 경고음과 복구 시도를 계속한다.

저장된 핸들이 닫혔거나 유효하지 않으면 폐기하고 실행 중인 허용 프로그램 또는 연결된 집중 WebView를 다시 탐색한다. 실행 중인 허용 대상이 하나도 없으면 경고음은 계속 내되 프로그램을 임의로 새로 실행하지 않는다.

## 구성 요소

### WindowManager

- 현재 전경 창 핸들과 프로세스 정보를 안전하게 조회한다.
- 창 핸들이 등록된 프로세스와 일치하는지 판정한다.
- 유효한 창을 복원하고 전면으로 가져오며 성공 여부를 반환한다.
- 프로세스 이름으로 실행 중인 복구 후보 창을 찾는다.

### Focus enforcement state

- 마지막 허용 창 핸들을 보관한다.
- 허용 상태에서는 복귀 후보를 갱신한다.
- 딴짓 상태에서는 경고와 복구가 필요한지를 결정한다.
- Windows API와 분리해 상태 전이를 단위 테스트할 수 있게 한다.

### StudyWindow

- 기존 1초 타이머에서 허용 여부와 현재 전경 창을 공용 구성 요소에 전달한다.
- 보조 FocusGuard 창과 외부 허용 창을 구분한다.
- 딴짓일 때 경고음을 재생하고 복구 명령을 수행한다.
- 원본과 로컬 전용 파일에 동일한 흐름을 유지한다.

## 오류 처리

- 프로세스가 종료되거나 창 핸들이 무효가 되는 경쟁 상태는 복구 실패로 처리하고 다음 타이머에서 다시 탐색한다.
- 접근 거부, 프로세스 종료, Windows API 실패는 집중 타이머를 중단시키지 않는다.
- 실패를 빈 `catch`로 숨기지 않고 디버그 진단에 남기되 사용자에게 매초 대화상자를 띄우지는 않는다.
- 경고음 재생 실패가 감시와 창 복구를 막지 않도록 분리한다.

## 테스트와 수동 검증

- 빈 입력칸에서 안내가 보이고 입력·삭제에 따라 사라졌다 다시 나타나는지 확인한다.
- 허용 앱 A에서 허용 앱 B로 이동한 뒤 딴짓 앱을 열면 B로 복귀하는지 확인한다.
- 마지막 허용 창이 최소화됐을 때 복원되는지 확인한다.
- 복귀 대상이 종료됐을 때 다른 허용 후보를 찾는지 확인한다.
- 복구가 실패하는 동안 매초 경고 요청이 반복되는지 단위 테스트한다.
- 휴식 중에는 경고와 강제 복구가 발생하지 않는지 확인한다.
- 원본 Apps Script 빌드와 로컬 번역 빌드를 각각 실행해 같은 동작인지 확인한다.

## 완료 조건

- 두 입력칸의 안내 문구가 실제 입력값으로 저장되지 않는다.
- 집중 중 허용되지 않은 앱으로 전환하면 경고음이 반복된다.
- 마지막으로 사용한 허용 프로그램 또는 집중 WebView가 자동 복원된다.
- 휴식 중에는 강제 복구가 비활성화된다.
- 원본과 로컬 프로젝트가 모두 빌드되고 동작이 일치한다.
